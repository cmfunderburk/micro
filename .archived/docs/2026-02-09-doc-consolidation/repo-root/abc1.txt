‚óŹ I've identified the bug. The issue is in RationalDecisionProcedure.choose():

  1. adjacent_agents returns a set[str] which has non-deterministic iteration order (depends on hash values)
  2. ProposeActions are added to the list in set iteration order
  3. max(evaluated, key=lambda x: x[1]) breaks ties by first occurrence
  4. Result: tie-breaking is non-deterministic, depending on Python's set iteration order

  The fix should add explicit lexicographic tie-breaking in choose(), consistent with the search module's approach. Let me implement this.
