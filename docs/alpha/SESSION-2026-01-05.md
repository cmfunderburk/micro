# Session Summary: 2026-01-05

## Core Engine Issues Resolved

### CE-3: Reproducibility (Critical) ✅
**Problem:** Global RNG and UUID-based agent IDs caused non-deterministic runs.

**Fix:**
- Added `_rng: Random` field to `Simulation` class
- `create_simple_economy()` creates seeded `Random(seed)` instance
- Agent IDs now deterministic format: `agent_000`, `agent_001`, etc.

**Files:** `simulation.py`, `agent.py`
**Tests:** `test_full_reproducibility` in `test_simulation.py`

---

### CE-1: Information Asymmetry (Critical) ✅
**Problem:** `NoisyAlphaInformation` existed but search/bargaining used true types, making info env behaviorally inert.

**Fix:**
- Search now uses `info_env.get_observable_type(target)` for target evaluation
- Removed protocol-dependent branching in surplus calculation
- Trade willingness based on observed types, not true types

**Files:** `search.py`
**Tests:** `test_noise_causes_different_target_valuations` in `test_information.py`

---

### CE-2: Self-Observation Noise (Medium) ✅
**Problem:** Observers applied noise to their own alpha when evaluating trades.

**Fix:**
- Observers use `AgentType.from_private_state()` for own type (no noise)
- Only targets observed through `info_env.get_observable_type()`

**Files:** `search.py`
**Tests:** `test_observer_knows_own_type` in `test_information.py`

---

### CE-4: Distance Metric Inconsistency (Medium) ✅
**Problem:** Perception used Euclidean distance, movement used Chebyshev. Agents could miss nearby targets that were close in travel time but far in Euclidean terms.

**Fix:**
- `agents_within_radius()` now uses Chebyshev distance
- Perception area is square (consistent with diagonal movement)

**Files:** `grid.py`, `search.py`, `simulation.py`
**Tests:** `test_agents_within_radius_uses_chebyshev`, `test_perception_movement_consistency` in `test_grid.py`

---

### CE-5: Wrapped Grid Chebyshev (Medium) ✅
**Problem:** `Position.chebyshev_distance_to()` ignored grid wrapping.

**Fix:**
- Added optional `grid_size` parameter to `chebyshev_distance_to()`
- Added `Grid.chebyshev_distance()` convenience method
- Search and simulation use wrapped Chebyshev on torus grids

**Files:** `grid.py`, `search.py`, `simulation.py`
**Tests:** `test_wrapped_chebyshev_distance` in `test_grid.py`
