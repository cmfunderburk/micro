**Key theorem:** Debreu's representation theorem (continuous preferences → continuous utility)

**Standard utility functions:**
- Cobb-Douglas: u(x,y) = x^α y^(1-α)
- CES: u(x,y) = (αx^ρ + (1-α)y^ρ)^(1/ρ)
- Linear: u(x,y) = αx + βy
- Leontief: u(x,y) = min(αx, βy)
- Quasi-linear: u(x,y) = x + v(y)

| Topic | Source | Reference |

**Bargaining protocols:**

1. **Take-it-or-leave-it (TIOLI):** Proposer captures all surplus beyond threat point. (O&R-B Ch 3.1)

2. **Nash Bargaining Solution:** Axiomatic—efficiency, symmetry, IIA, scale invariance. Solution: argmax (u₁ - d₁)(u₂ - d₂) subject to feasibility. (O&R-B Ch 2, Kreps II Ch 23)

3. **Rubinstein Alternating Offers:** Finite horizon uses backward induction; infinite horizon has unique SPE; δ → 1 converges to Nash. Key insight: patience = bargaining power. (O&R-B Ch 3, O&R-G Ch 7)

**Key theorems:**
1. **Nash (1950):** Unique bargaining solution satisfying four axioms
2. **Rubinstein (1982):** Unique SPE of alternating-offer game; converges to Nash as δ→1
3. **Core convergence:** As economy replicates, core shrinks to Walrasian equilibrium

### Mechanism Design and Implementation

| Topic | Source | Reference |
|-------|--------|-----------|
| Revelation principle | Kreps II | Ch 26, Appendix 16 |
| Gibbard-Satterthwaite | Kreps II | Ch 26 |
| VCG mechanisms | Kreps II | Ch 26 |
| Implementation theory | Kreps II | Ch 26 |
| Implementation (game-theoretic) | O&R-G | Ch 10 |
| Nash implementation | O&R-G | Ch 10.2 |
| Dominant strategy implementation | O&R-G | Ch 10.3 |
| Complexity considerations | O&R-G | Ch 9 |

**Key concepts:**

1. **Revelation Principle:** Any outcome achievable by a mechanism can be achieved by a direct mechanism where truthful reporting is an equilibrium. (Kreps II Ch 26, Appendix 16)

2. **Implementation:** Given a social choice correspondence, find a game form whose equilibrium outcomes coincide with the desired outcomes for all preference profiles. The planner designs the rules; players' preferences and actions remain private. (O&R-G Ch 10)

3. **Nash vs. Dominant Strategy Implementation:** Nash implementation requires less restrictive conditions but depends on common knowledge of preferences; dominant strategy implementation is more robust but more restrictive. (O&R-G Ch 10.2-10.3)

**Key theorems:**
1. **Gibbard-Satterthwaite:** No non-dictatorial voting mechanism is strategy-proof for unrestricted preferences
2. **Maskin (1999):** Monotonicity is necessary for Nash implementation; with no veto power, it's sufficient
3. **VCG:** Efficient allocation with dominant strategy incentive compatibility when transfers are allowed
